Aggressive change kept:
- Added an inline Triton kernel in `submission.py` that reproduces the public AITER MXFP4 quantization path while emitting shuffled scale layout directly.
- Replaced:
  - `dynamic_mxfp4_quant(A)`
  - `e8m0_shuffle(...)`
- With:
  - `_quant_mxfp4_shuffled_inline(A)`
- Kept the current best GEMM dispatch table exactly the same:
  - `M < 32` -> direct ASM
  - exact `(M=32, N=4096, K<=1024)` -> direct ASM `32x128`
  - exact `(M=32, N=2880, K<=1024)` -> direct ASM `64x128`
  - everything else -> `aiter.gemm_a4w4(...)`

Why this works:
- Before iteration 7, the hot path for `A` quantization did two GPU-side steps:
  - quantize `A`
  - reorder/shuffle scale metadata
- The inline Triton path folds those into one quantization kernel that writes the shuffled scale layout directly.
- That removes an extra tensor transform and a full extra pass over the scale tensor.
